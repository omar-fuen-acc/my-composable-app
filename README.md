# my-composable-app

A REST API built with [Mercury Composable](https://github.com/Accenture/mercury-nodejs) — an event-driven, composable microservices framework for Node.js.

## Requirements

- Node.js >= 20.18.1
- npm

## Setup

Install dependencies:

```bash
npm install
```

## Build

Compile TypeScript and copy resource files to `dist/`:

```bash
npm run build
```

> Run this command every time you make changes to the source code before starting the server.

To clean the `dist/` folder:

```bash
npm run clean
```

## Run

```bash
npm start
```

The server starts on **http://localhost:8086** by default.

## Endpoints

### GET /api/hello

Returns a Hello World greeting.

**Request:** No parameters required.

**Response:**
```json
{
  "message": "Hello, World!"
}
```

---

### GET /api/products

Returns the full product catalogue.

**Request:** No parameters required.

**Response:**
```json
{
  "products": [
    {
      "item_id": 1,
      "item_name": "...",
      "item_category": "...",
      "item_price": 0.00,
      ...
    }
  ]
}
```

---

### GET /api/product/{id}

Returns a single product by its `item_id`.

**Path parameter:**

| Parameter | Type   | Description                  |
|-----------|--------|------------------------------|
| `id`      | number | The `item_id` of the product |

**Responses:**

- `200 OK` — product object found
- `302 Redirect` → `/api/products` — no product matches the given `id`

**Example:**
```
GET /api/product/3
```

---

### POST /api/addToCart

Adds one or more products to a user's shopping cart. If the user doesn't have a cart yet, a new one is created. Duplicate items (already in the cart) are ignored.

**Request body** (`Content-Type: application/json`):

| Field     | Type       | Required | Description                          |
|-----------|------------|----------|--------------------------------------|
| `user_id` | string     | Yes      | Unique identifier for the user       |
| `items`   | number[]   | Yes      | Array of `item_id` values to add     |

**Example body:**
```json
{
  "user_id": "user-123",
  "items": [1, 4, 7]
}
```

**Responses:**

- `200 OK` — returns the user's updated cart
- `400 Bad Request` — missing or invalid `user_id` or `items`
- `404 Not Found` — none of the provided item IDs exist in the catalogue

**Example response:**
```json
{
  "user_id": "user-123",
  "items": [
    { "item_id": 1, "item_name": "...", ... },
    { "item_id": 4, "item_name": "...", ... }
  ]
}
```

---

### GET /api/shoppingList

Returns the shopping cart for a given user.

**Query parameter:**

| Parameter | Type   | Required | Description                    |
|-----------|--------|----------|--------------------------------|
| `user_id` | string | Yes      | The user whose cart to retrieve |

**Example:**
```
GET /api/shoppingList?user_id=user-123
```

**Responses:**

- `200 OK` — returns the user's cart object
- `400 Bad Request` — `user_id` query parameter is missing
- `404 Not Found` — no cart exists for the given `user_id`

**Example response:**
```json
{
  "user_id": "user-123",
  "items": [
    { "item_id": 1, "item_name": "...", ... }
  ]
}
```

## Project Structure

```
my-composable-app/
├── src/
│   ├── main.ts                   # Entry point
│   ├── preload/
│   │   └── preload.ts            # Bootstrap: registers all composables
│   ├── functions/                # Composable function implementations
│   │   ├── hello-world.ts
│   │   ├── get-products.ts
│   │   ├── get-product-by-id.ts
│   │   ├── add-to-cart.ts
│   │   └── get-shopping-list.ts
│   ├── resources/
│   │   ├── rest.yaml             # HTTP route definitions
│   │   ├── flows.yaml            # Flow registry
│   │   └── flows/                # Event-driven flow definitions (YAML)
│   │       ├── hello-world.yml
│   │       ├── get-products.yml
│   │       ├── get-product-by-id.yml
│   │       ├── add-to-cart.yml
│   │       └── get-shopping-list.yml
│   └── mock/
│       ├── products.json         # Product catalogue (read-only)
│       └── shoppingCarts.json    # Persisted shopping carts (read/write)
├── dist/                         # Compiled output (generated by build)
├── copy-resource-files.js        # Build helper: copies resources and mock data
├── clean.js                      # Build helper: removes dist/
├── package.json
└── tsconfig.json
```
